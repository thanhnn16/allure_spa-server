ERD:
1. Users:
   - Thuộc tính chính: id (PK), phone_number, email, role
   - Mối quan hệ:
     * 1:N với Addresses
     * 1:1 với Staff_Details
     * 1:N với Orders
     * 1:N với Invoices
     * 1:N với User_Treatment_Packages
     * 1:N với Appointments
     * 1:N với Ratings
     * 1:N với Favorites
     * 1:N với Notifications
     * 1:N với FCM_Tokens
     * 1:N với Chats (as user)
     * 1:N với Chats (as staff)
     * 1:N với Employee_Attendance
     * 1:N với User_Vouchers

2. Addresses:
   - Thuộc tính chính: id (PK), user_id (FK), address, address_type
   - Mối quan hệ: N:1 với Users

3. Staff_Details:
   - Thuộc tính chính: user_id (PK, FK), position, hire_date
   - Mối quan hệ: 1:1 với Users

4. Products:
   - Thuộc tính chính: id (PK), name, price, category_id (FK), image_id (FK)
   - Mối quan hệ:
     * N:1 với Product_Categories
     * 1:1 với Product_Details
     * 1:N với Product_Price_History
     * 1:N với Product_Attributes
     * 1:N với Stock_Movements
     * 1:N với Product_Translations
     * N:M với Attributes (thông qua Product_Attributes)
     * 1:N với Order_Items
     * 1:N với Cart_Items

5. Product_Categories:
   - Thuộc tính chính: id (PK), category_name, parent_id (FK)
   - Mối quan hệ:
     * 1:N với Products
     * 1:N với Product_Category_Translations
     * 1:N với chính nó (quan hệ cha-con)

6. Treatments:
   - Thuộc tính chính: id (PK), name, price, category_id (FK), image_id (FK)
   - Mối quan hệ:
     * N:1 với Treatment_Categories
     * 1:N với Treatment_Price_History
     * 1:N với Treatment_Combos
     * 1:N với Appointments
     * 1:N với Ratings
     * 1:N với Treatment_Translations
     * 1:N với User_Treatment_Packages
     * 1:N với Order_Items
     * 1:N với Cart_Items

7. Treatment_Categories:
   - Thuộc tính chính: id (PK), category_name, parent_id (FK)
   - Mối quan hệ:
     * 1:N với Treatments
     * 1:N với Treatment_Category_Translations
     * 1:N với chính nó (quan hệ cha-con)

8. Invoices:
   - Thuộc tính chính: id (PK), user_id (FK), staff_user_id (FK), total_amount
   - Mối quan hệ:
     * N:1 với Users (as customer)
     * N:1 với Users (as staff)
     * 1:N với Invoice_Payments
     * 1:N với Payment_Histories
     * 1:N với Treatment_Usage_History

9. Orders:
   - Thuộc tính chính: id (PK), user_id (FK), total_amount
   - Mối quan hệ:
     * N:1 với Users
     * 1:N với Order_Items

10. Carts:
    - Thuộc tính chính: id (PK), user_id (FK)
    - Mối quan hệ:
      * N:1 với Users
      * 1:N với Cart_Items

11. Vouchers:
    - Thuộc tính chính: id (PK), code, discount_type, discount_value
    - Mối quan hệ:
      * 1:N với User_Vouchers
      * 1:N với Voucher_Translations

12. Reward_Items:
    - Thuộc tính chính: id (PK), name, item_type, item_id, points_required
    - Mối quan hệ:
      * 1:N với Point_Redemption_History
      * 1:N với Reward_Item_Translations

13. Appointments:
    - Thuộc tính chính: id (PK), user_id (FK), treatment_id (FK), staff_user_id (FK), start_time
    - Mối quan hệ:
      * N:1 với Users (as customer)
      * N:1 với Treatments
      * N:1 với Users (as staff)

14. User_Treatment_Packages:
    - Thuộc tính chính: id (PK), user_id (FK), treatment_id (FK), total_sessions, used_sessions
    - Mối quan hệ:
      * N:1 với Users
      * N:1 với Treatments
      * 1:N với Treatment_Usage_History

15. Ratings:
    - Thuộc tính chính: id (PK), user_id (FK), treatment_id (FK), stars
    - Mối quan hệ:
      * N:1 với Users
      * N:1 với Treatments

16. Favorites:
    - Thuộc tính chính: id (PK), user_id (FK), favorite_type, item_id
    - Mối quan hệ: N:1 với Users

17. Notifications:
    - Thuộc tính chính: id (PK), user_id (FK), content
    - Mối quan hệ: N:1 với Users

18. Chats:
    - Thuộc tính chính: id (PK), user_id (FK), staff_id (FK)
    - Mối quan hệ:
      * N:1 với Users (as user)
      * N:1 với Users (as staff)
      * 1:N với Chat_Messages

19. Employee_Attendance:
    - Thuộc tính chính: id (PK), user_id (FK), check_in, check_out
    - Mối quan hệ: N:1 với Users

20. Banners:
    - Thuộc tính chính: id (PK), image_url, link

21. Images:
    - Thuộc tính chính: id (PK), image_path
    - Mối quan hệ:
      * 1:N với Users
      * 1:N với Products
      * 1:N với Treatments

22. Attributes:
    - Thuộc tính chính: id (PK), attribute_name
    - Mối quan hệ:
      * N:M với Products (thông qua Product_Attributes)
      * 1:N với Attribute_Translations

23. Treatment_Usage_History:
    - Thuộc tính chính: id (PK), user_treatment_package_id (FK), user_id (FK), treatment_id (FK), staff_user_id (FK), invoice_id (FK)
    - Mối quan hệ:
      * N:1 với User_Treatment_Packages
      * N:1 với Users (as customer)
      * N:1 với Treatments
      * N:1 với Users (as staff)
      * N:1 với Invoices

24. User_Vouchers:
    - Thuộc tính chính: user_id (FK), voucher_id (FK)
    - Mối quan hệ:
      * N:1 với Users
      * N:1 với Vouchers

25. Order_Items:
    - Thuộc tính chính: id (PK), order_id (FK), item_type_id, item_id
    - Mối quan hệ:
      * N:1 với Orders
      * N:1 với Products (khi item_type là product)
      * N:1 với Treatments (khi item_type là treatment)

26. Cart_Items:
    - Thuộc tính chính: id (PK), cart_id (FK), item_type_id, item_id
    - Mối quan hệ:
      * N:1 với Carts
      * N:1 với Products (khi item_type là product)
      * N:1 với Treatments (khi item_type là treatment)

Lưu ý: PK là Khóa Chính, FK là Khóa Ngoại
